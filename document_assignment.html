Congratulations, you have landed your first job doing "Web Design." Your client is planning on making a competitor for Google Docs and Microsoft Office 365. They have created an engine that spots spelling and grammar problems in 2 categories: errors and suggestions. As well as auto-detects quotable paragraphs. These things are marked within the text, using css classes.
Normally, using newer web standards, some of these things could be addressed using existing tags, and not handled manually, but for some reason the back-end developer thought that the only tag that should ever be needed is div. Without fixing the auto-generated text (so, from a separate stylesheet called document_example.css below we need you to:
  1. Underline all errors with that wavy underline all the cool word processors are using these days
  2. Underline all suggestions with straight underlines
  3. Make spelling underlines red
  4. Make grammar underlines blue
  5. Do not underline things in blockquotes.
  6. Put a rounded (5px radius)lightgray background around blockquotes
  7. Give blockquotes 10px of padding
  8. put a 20px margin on the left and right of blockquotes
  9. blockquotes within blockquotes should not alter the text, nested blockquotes should have no effect.
  10. Ensure that other tags do not interrupt the flow of the text
  11. Do not display tags with the ignore class
  12. Headings in the main document need to be bold and have a font size of 3em
  13. Headings not in the main document (headings that are one level down) need to appear on their own line and be bold
  14. Set the body width to 800px and body margin to auto to center the body.
  15. Finally, hide this paragraph, upload this along with the rest of your site using filezilla, link to this page from your main page, and turn in a link to this page
